🗺 Regra de Agrupamento de Bens
O ID do bem segue uma lógica de ciclo: sempre que o sufixo termina em 01, inicia-se um novo ativo (bem-pai). Os IDs subsequentes são componentes filhos.
RSSNA01 → Bem-pai (embarcação)
RSSNA02 → Motor Propulsão 1
RSSNA03 → Motor Propulsão 2
RSSNA04 → Motor Auxiliar 1
…
RSCDO01 → Novo bem-pai
RSCDO02 → Motor Propulsão 1
…

ROTEIRO DE IMPORTAÇÃO · SIGAMNT
5 etapas sequenciais para configurar o plano completo no Protheus 12 — a ordem é obrigatória
📌 Sequência de Configuração
Cada etapa depende da anterior — não pular
1
Cadastrar Grupos / Tarefas
Criar os 93 grupos na tabela SM8 usando o código TAR-XXX-NNN. O grupo deve existir ANTES de qualquer instrução ser importada. Registros de instrução sem grupo válido são silenciosamente rejeitados.
Aba: GRUPOS_TAREFAS Tabela: SM8
2
Cadastrar Instruções / Etapas
Importar as 320 instruções (INS001–INS320) na tabela SM7, vinculando cada uma ao grupo correspondente. Configurar frequência em dias: Semanal=7, Mensal=30, Semestral=180, Anual=365.
Aba: INSTRUCOES_MASTER Tabela: SM7
3
Cadastrar Bens (Ativos)
Criar os 85 bens na tabela SM2 respeitando a hierarquia: ID terminado em "01" = bem-pai. Bens filhos devem referenciar o pai. A frota RS AREIA BRANCA (11 bens) ainda não está no sistema.
Aba: BENS_CADASTRO Tabela: SM2 / SIGAATF
4
Montar Roteiros de Manutenção
Criar 1 roteiro por embarcação na tabela SM5, vinculando Bem + Grupos + Instruções. Nomenclatura: ROT-[ID_BEM]-PREV. Componentes críticos (motor, leme) podem ter roteiros próprios.
Nomenclatura: ROT-RSSNA01-PREV Tabela: SM5
5
Configurar Geração de OS
Para motores: gatilho por horímetro (MCP=400h, MCA=500h, GER=250h). Para demais sistemas: gatilho por data/calendário. Configurar perfis distintos: executor faz apontamento; supervisor fecha e aprova.
Horímetro nos bens de motor Tabela: SM0/SM1
⚠ Alertas Críticos de Implantação
🚫 Rejeição Silenciosa de Registros
Instruções importadas sem grupo válido na SM8 não geram erro — são simplesmente ignoradas. Sempre validar grupos antes de importar etapas.
🚫 OS Vazia por Roteiro sem Instrução
Roteiros sem instruções vinculadas geram OS que não travam o fluxo mas corrompem o histórico de execução e invalidam auditoria ISO 9001.
⚠ Perfil Executor = Supervisor
O erro mais comum: usar o mesmo perfil para quem executa e quem aprova. O executor passa a fechar OS sem aprovação, eliminando o controle de qualidade.
⚠ Patches do Protheus
Atualizações de release podem redefinir parâmetros SX3 e periodicidades. Exportar roteiros e planos ANTES de qualquer LP/patch e validar em ambiente de homologação.
